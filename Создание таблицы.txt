Создание таблицы производится следующим образом:

CREATE TABLE person
  (person_id SMALLINT UNSIGNED,
    fname VARCHAR(20),
    lname VARCHAR(20),
    eye_color CHAR(2),
    birth_date DATE,
    street VARCHAR(30),
    city VARCHAR(20),
    state VARCHAR(20),
    country VARCHAR(20),
    postal_code VARCHAR(20),
    CONSTRAINT pk_person PRIMARY KEY (person_id)
);

Наиболее примечательно тут то, что первичный ключ задается вручную. Делается это в результате создания ограничения таблицы, ограничение первичного ключа.
В определение таблицы можно добавить несколько ограничений. В данном случае первичный ключ создается в столбце person_id и получает имя pk_person.

Другой тип ограничений называется проверочным. Необходим для того, чтобы проверять входные значения прежде, чем сохранить их в таблице.
Так выглядит проверочное ограничение для столбца eye_color:
eye_color CHAR(2) CHECK (eye_color in 'BR', 'BL', 'GR')),

Обычно проверочное ограничение применяется к данным и работает должным образом, но в MySQL это не так. С целью фильтрации строковых данных, попадающих в таблицу, возможно применять применять тип данных, именуемый ENUM.
eye_color ENUM ('BR', 'BL', 'GR'),
