insert имеет три основных компонента:
- имя таблицы;
- имена столбцов, в которые будут заноситься данные;
- значения, которые будут занесены

Заполнять данными обязательно лишь те колонки, которые определены как not null.

Генерация данных числовых ключей.
Для того, чтобы сгенерировать первичный числовой ключ, имеется два способа:
1) сгенерировать его в результате инкрементации самого большого значения на некое число;
2) позволить СУБД самой решить эту проблему.

Первый вариант плох из- за возможных коллизий. Поэтому на практике применяется второй вариант.

В Oracle Database существует отдельный объект схемы, называемый последовательностью. В случае MySQL необходимо включить функцию автоинкрементирования.

Обычно такая фишка задается при создании таблицы, но если этого не произошло, следует поступить следующим образом: 
ALTER TABLE 'название_таблицы' MODIFY 'определение_колонки (person_id SMALLINT UNSIGNED AUTO_INCREMENT)';

Приведенная выше функция переопределяет столбец person_id, теперь достаточно заносить null в него, а MySQL самостоятельно подставит вычесленное значение
